MitApp Inventor Project

QuizMe App
Play the Game!

In this pair-programming project my partner and I took an existing tutorial from MIT App Inventor and refined it, improving the game stylistically and interface-wise. The changes that we added into the game itself was a score system, multiple choice answers, and refining the game to make it flow easier. Stylistically we added more questions that would appeal to the newer generations, flashy text, home and finishing screens, and changing the overall layout to make it all fit on one screen without scrolling. From this project I learned many important skills for the coding industry. I learned about documenting all the individual versions of the game. Whenever we completed a big task, we would save it as a version of the game and then continue with our next task. If the new task fails, we can always just restart with the saved version. Along with that, while pair programming we couldn’t work on it at the same time. To compensate for this we would have individual test versions that we would work on by ourselves and then once it was complete we would add it to the main version. This required a high level of communication in order to be the most time efficient.

Game Theory Project

Prisoner's Dilema

Prisoner’s Dilemma is a game involving at least two teams. The idea is that prisoners are being separately questioned and are either gonna rat the other player out (betray), or lie and keep it a secret (collude). For every round, each team must decide whether they will collude or betray. Both colluding will result in 0 points. You betraying while they collude means you gain 100 points. Them betraying but you colluding results in -500 points. And finally, both betraying results in -250 points. This game is played for a number of rounds and at the end, the team with the highest score wins. My partner and I investigated the code that was running the game and it shows that the scores are calculated at the end. Essentially what we did was change our history according to what our opponents had inputted before. Although we did not win, we did relatively well and our code was a success. Through this project, I learned more about interacting using GItHub as well as team skills like collaboration and patience.
